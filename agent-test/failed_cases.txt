## 失败用例详情

### 1. Python venv 创建失败 (2个)
**测试名称**: 创建venv并安装包
**输入命令**: python3 -m venv myenv && myenv/bin/pip install requests
**输出**:
```
The virtual environment was not created successfully because ensurepip is not
available.  On Debian/Ubuntu systems, you need to install the python3-venv
package using the following command.

    apt install python3.13-venv
```
**失败原因**: 容器缺少 python3-venv 包

---

### 2. Shell whoami 失败
**测试名称**: 用户信息
**输入命令**: whoami
**输出**:
```
whoami: cannot find name for user ID 99999: No such file or directory
```
**失败原因**: UID 99999 在 /etc/passwd 中没有对应条目（预期行为，非 bug）

---

### 3. Shell case 语句失败
**测试名称**: Shell case语句
**输入命令**: x=2; case $x in 2) echo two;; esac
**输出**: (空)
**失败原因**: 测试用例逻辑问题或变量传递问题

---

### 4. Node.js HTTP 请求被杀死
**测试名称**: Node.js HTTP请求
**输入命令**: node -e "require('http').get('http://example.com', r => console.log(r.statusCode))"
**输出**: 退出码 137 (被杀死)
**失败原因**: 可能是超时或内存限制

---

### 5. Node.js HTTPS 证书验证失败
**测试名称**: Node.js HTTPS请求
**输入命令**: node -e "require('https').get('https://example.com', r => console.log(r.statusCode))"
**输出**:
```
Error: unable to get local issuer certificate
code: 'UNABLE_TO_GET_ISSUER_CERT_LOCALLY'
```
**失败原因**: SSL 证书验证失败，缺少 CA 证书
